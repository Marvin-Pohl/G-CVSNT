Установка клиентской части программы TortoiseCVS - простейшая интеграция SSH в Windows

Описание: Как получить упавление версиями прямо в Проводнике Windows (Windows Explorer), включая внедрение SSH.

Инструкции по установке клиента TortoiseCVS

Данный документ предоставляет инструкции для установки и использования клиентской части CVS - TortoiseCVS. Эти инструкции созданы для разработчиков проектов размещенных на SourceForge.net, которые используют 32-битную платформу Microsoft Windows. Для получения предварительной информации вы должны прочитать другие доступные документы о CVS.

TortoiseCVS имеет преймущества по сравнению с WinCVS для работы с SourceForge. Он имеет простой интерфейс который интегрирован в Windows Explorer. WinCVS имеет более сложный и высокоразвитый интерфейс. Для более сложных команд может быть проще или необходимо использовать WinCVS.

Особая выгода состоит в том, что TortoiseCVS имеет встроенного клиента SSH, без специальной настройки. Любой может легко помещать код в SourceForge.net проекты используя TortoiseCVS.

Установка TortoiseCVS

  1. Зайдите на tortoisecvs.org и загрузите версию 1.6.14 или более позднюю стабильную версию со страницы загрузки (Download).
  2. Для установки, запустите выполняемый файл и следуюйте инструкциям (на английском).
  3. Так как TortoiseCVS это расширение оболочки, вы нуждаетесь в перезагрузке вашей машины.

Извлечение (Checking out) существующего проекта

Если вы работаете над существующим проектом, сперва вы нуждаетесь в извлечении кода.

  1. Убедитесь что вы имеете учетную запись (account) в SourceForge.net, и что вы были добавлены как разаработчик к проекту в котором вы хотите учавствовать. Это должны делать администраторы проекта.
  2. В Проводнике (Windows Explorer) перейдите в папку в котрорую вы хотите извлечь код.
  3. Щелчек правой кнопкой мыши и из контекстного меню выберите команду "CVS Checkout", появится диалог.
  4. Вы должны указать ваше пользовательское имя (логин) на SourceForge.net, и UNIX имя проекта. Заполните диалог в следующей последовательности:
    • Protocol: Secure Shell
    • Server: cvs.sourceforge.net
    • Directory: /cvsroot/имя проекта
    • Username: логин на sourceforge.net
  5. Нажмите "Fetch List" для получения списка модулей. Вы должны будете ввести свой пароль в диалоге. Затем выберите модуль, который вам нравится.
  6. Нажмите OK. TortoiseCVS начнет получать код. Вы снова должны ввести ваш пароль.
  7. Сейчас вы можете изменять файлы и затем выбрать "CVS Commit" в директории верхнего уровня для сохранения ваших изменений в репозитарии. Для более подробной информации, прочитайте Руководство по ежедневному использованию TortoiseCVS.

Получаете ошибку? Если вы получаете ошибку говорящую что-то подобное "Could not chdir to home directory /home/users/r/rd/rdonkin: No such file or directory" тогда вы должны войти на сервер с использованием SSH для создания вашей домашней папки. Это делается соединением с username@cvs.sf.net через одно из двух: Putty или командной строки SSH.

Работа над новым проектом

Если вы работаете над новым проектом, тогда вы сначала должны создать новый модуль в репозитории.

  1. Создайте папку с именем которое должен иметь модуль.
  2. Поместите файлы которые вы ходите добавить в папку.
  3. Щелкните Правым щелчком на директории, и выберите Make New Module из подменю CVS.
  4. Заполните диалог, как описано в предыдущей секции. Обратите внимание что имя модуля заполняется вами.
  5. Нажмите OK. Вы будете должны дважды ввести ваш пароль, и модуль будет создан.
  6. Продолжайте добавлять файлы и папки правыми щелчками мышью и выбором "CVS Add" или "CVS Add All Recursively" из контекстного меню.
  7. Вы можете сейчас поместить все файлы, правым щелчком на папке верхнего уровня и выбрав "CVS Commit".
  8. Для более подробной информации, прочитайте Руководство по ежедневному использованию TortoiseCVS.

Надоело вводить ваш пароль все время?

Вы должны сделать пару ключей публичный/личный. Публичный ключ поместите на сервер, личный ключ расположите на вашем жестком диске. Сейчас, когда вы соеденяетесь сервер может проверить кто вы, так как только вы имеете ваш личный ключ. можете проверить кто

Для дальнейшей защиты вашего личного ключа, он зашифрован на вашем жестком диске с помошью ключевой фразы. Вы можете однако выбрать пустую ключевую фразу, в этом случае вы никогда не вводите пароль, но любой кто имеет доступ к вашему компьютеру, может использовать вашу SSH CVS учетную запись . Либо вы можете использовать агента SSH (например Pageant) который запоминает вашу ключевую фразу во время одной сессии, так что вы вводите пароль один раз.

TortoiseCVS использует версию Plink как собственный клиент SSH. Это означает что вы можете использовать PuTTYgen для генерации пар публичных/личных ключей, и вы можете использовать Pageant для указания где располагаются личные ключи. Сначала загрузите все с сайта PuTTY.

Для информации об их использовании прочитайте в документации PuTTY documentation об PuTTYgen и Pageant. Особено убедитесь что прочитали раздел "8.2: Using PuTTYgen, the PuTTY key generator" (Использование генератора ключей PuTTYgen) и "9.3.1 Making Pageant automatically load keys on startup" (настройка Pageant для автоматической загрузки ключей во время старта).

После изготовления вашего публичного ключа с помощью PuTTYgen вы нуждаетесь в его выгрузке используя веб интерфейс SourceForge.net. Войтите на веб-сайт SourceForge, перейдите на вашу страницу Account Preferences (предпочтания учетной записи). Пролистайте в низ до раздела "Shell Account Information" в котором есть параметр "CVS/SSH Shared Authorized Keys". Здесь вы должны добавить ваш публичный ключ.
